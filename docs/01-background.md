# 项目背景与问题定义

## 业务场景

某中型电商平台（模拟数据），月度活跃用户约 5,000 人，月订单量约 2,000 单。运营团队面临以下问题：

### 问题 1：运营资源分配低效

当前对所有注册用户无差别推送营销信息。结果是：高价值用户收到与普通用户相同的促销内容（触达不足），低价值/已流失用户持续收到推送（资源浪费）。**月营销预算 20,000 元，但无法评估投放效率。**

### 问题 2：品类推荐缺乏差异化

运营团队按"热销排行"统一推送商品，所有用户收到的推荐相同。但直觉上，不同消费习惯的用户应该有不同的品类偏好——**需要数据验证并量化这些差异。**

## 项目目标

1. **将用户划分为可独立运营的客群**（每组有明确的特征画像和差异化策略）
2. **量化各客群的品类偏好差异**（让品类推荐有数据依据）
3. **输出可执行的触达清单**（每个用户 → 运营动作 → 预估响应率）
4. **预估每组运营 ROI**（让预算分配有数据支撑）

## 成功标准

| 维度 | 标准 |
|------|------|
| 分群覆盖 | 100% 用户有明确的客群归属 |
| 分群可运营 | 每组 ≥ 16 人（统计有意义的最小样本），内部行为一致性高 |
| 品类洞察 | 至少 3 个组有明确区别于均值的品类偏好（TGI > 120 或 TGI < 80） |
| 策略可执行 | 每个组有 ≥ 1 条区别于其他组的差异化运营动作 |
| ROI 可量化 | 整体预估 ROI > 3:1 |
