# 方法论：为什么这样设计

## RFM 阈值选择依据

### R（Recency）：60天 / 180天

通过分析 R 分布的多组阈值对比（30/90, 60/120, 60/180, 90/180, 90/365），选择 60/180 的理由：

- 60 天约覆盖活跃用户的 40-50%，是电商常见的"活跃"定义（2个月内买过）
- 180 天约覆盖到 70-75%，超过半年的用户回购概率急剧下降
- 60/180 产生的三组人数分布相对均衡（约 20% / 30% / 50%），避免极端不均衡

### F（Frequency）：2 档而非 3 档

数据显示约 95% 用户仅购买 1 次：
- 1 次：~95%
- 2 次：~3%
- 3+ 次：~2%

如果 F 分 3 档（如 1次 / 2-3次 / 4+次），第 2、3 档几乎为空，无法支撑独立运营。F 分 2 档是**数据驱动的决策**，不是方法论上的简化。

### M（Monetary）：P25/P75 分位数 + 箱线图上边缘

M 分布右偏（均值 > 中位数，存在长尾高消费用户），等距划分会导致大部分用户集中在低档。P25/P75 分位数法确保每档有足够用户。

VIP 使用箱线图上边缘（Q3 + 1.5×IQR）——这是一个**统计异常检测方法**，用于识别消费金额远超正常范围的用户。这一定义的特点是：VIP 识别不受 R 和 F 影响，因此会包含"单次大额消费后流失"的用户。

### 局限性声明

- VIP 定义偏重 M，可能混淆"大额一次性消费"和"持续高消费"
- 生产环境中建议 VIP 增加 R 和 F 约束（如 R<60 + F≥2 + M>P90）
- R 阈值应根据实际品类购买周期调整（当前为通用电商阈值）
- 阈值应定期（每季度）重新评估和调整

## 为什么从 7 组拆分为 10 组

原 7 组中有 3 个组内部差异过大：

| 原组 | 人数 | 内部差异 | 拆分结果 |
|------|------|---------|---------|
| 唤醒沉睡人群 | 530 (29%) | 高价值复购沉睡 vs 低价值一次性沉睡 | → 唤醒高潜 + 唤醒普通 |
| 活跃一次性新客 | 275 (15%) | 高消费首单 vs 低消费首单 | → 高潜新客 + 普通新客 |
| 流失低价值低效 | 765 (42%) | 42%人在一组无法执行统一策略 | → 流失可弃 + 流失低潜 |

拆分原则：每组够 16 人（统计最小样本）+ 组内行为模式一致 + 可执行单一策略。

## 为什么用 TGI 替代绝对金额排名

原始分析中所有组的 TOP3 品类都是 Electronics > Automotive > Home & Kitchen——这是因为 Electronics 在所有品类的绝对销售额中占比最高（约 40%），不是每个组"偏好"Electronics。

TGI（Target Group Index）通过标准化消除品类结构的影响：

```
TGI = (组在品类A的消费占比) / (全体在品类A的消费占比) × 100

TGI = 120 → 该组在品类A上的消费集中度比全体高 20%
TGI = 80  → 该组在品类A上的消费集中度比全体低 20%
```

TGI 方法适用于任何"想要发现相对偏好而非绝对大小"的分析场景。
