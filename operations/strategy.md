# 10大运营组差异化策略文档

> 基于 RFM 分群 + TGI 品类偏好分析的精准运营方案.

---

## 分析方法论

采用自下而上的分析流程：实际数据分布 → 确定 RFM 阈值 → 19类基础分群 → 10大运营组 → TGI品类偏好 → 差异化策略。

**为什么是10组（而非原7组）**：原7组中"唤醒沉睡人群"（内部包含高潜复购和低潜一次性）、"活跃一次性新客"（高消费和低消费混在一起）、"流失低价值低效"（42%用户无法执行统一策略）三个组内部差异过大。拆分为10组后，每组可执行独立策略。

**TGI 偏好分析**：使用 TGI（Target Group Index）替代绝对金额排名。TGI > 120 表示该组在该品类的消费占比显著高于全体均值。

## ROI 预估方法论

### 响应率基准

| 渠道                          | 基准响应率 | 单次成本     | 适用客群           |
| ----------------------------- | ---------- | ------------ | ------------------ |
| 邮件-个性化（含动态商品推荐） | 15-30%     | ¥0.50       | VIP/高价值         |
| 邮件-定向（分段内容）         | 8-15%      | ¥0.15       | 活跃/唤醒          |
| 邮件-群发                     | 2-5%       | ¥0.10       | 流失/低潜          |
| SMS                           | 5-15%      | ¥0.08       | 流失高潜（双触达） |
| 专属客服外呼                  | 25-50%     | ¥25.00      | VIP（仅限）        |
| 优惠券核销                    | 10-25%     | 面额×核销率 | 所有组             |

> 基准来源：Campaign Monitor 2025 电商行业报告 / Klaviyo Ecommerce Benchmarks 2025

### 响应率乘数模型

不同客群状态对同一渠道的响应率有显著差异，通过乘数调整：

```
实际响应率 = 渠道基准 × R乘数 × F乘数 × M乘数

乘数参考（定性估计，来自行业文献中 RFM 各维度对响应率影响的定性结论，
而非本项目数据的统计回归——本项目使用模拟数据，不含历史营销响应记录）：
  R乘数: ≤60天×1.5, 61-180天×0.6, >180天×0.3
        依据：最近活跃用户的响应率通常为流失用户的 3-5 倍（Marketing Metrics, Farris et al., 2015），本项目取 5 倍（1.5/0.3）
  F乘数: 1次×0.7, ≥2次×1.0
        依据：复购用户邮件打开率/点击率比首次购买者高约 30-40%（Klaviyo 2025 基准）
  M乘数: 低×0.7, 中×1.0, 高×1.3
        依据：高消费用户对促销的响应意愿通常更高（营销分层定价文献），但差异程度缺乏一致的定量估计
```

> **注意**：上述乘数为启发式估计（heuristic estimates），非本项目数据驱动校准。在实际运营中需要通过 AB 测试或多臂老虎机实验进行校准。在获得校准数据之前，三档敏感性分析已覆盖乘数不确定性的主要影响。

**计算示例**——流失高潜用户 × 邮件定向：

- 邮件定向基准 8%
- R>180天 ×0.3 → 2.4%
- F≥2 ×1.0 → 2.4%
- M中高 ×1.3 → **3.1%**（比直觉估计的10%保守得多）

### 优惠券成本模型

| 客群层级   | 面额范围 | 核销率假设 |
| ---------- | -------- | ---------- |
| VIP/高价值 | ¥10-20  | 70%        |
| 活跃用户   | ¥20-30  | 50%        |
| 唤醒用户   | ¥30-50  | 30%        |
| 流失用户   | ¥50-100 | 15%        |

### 增量收入假设

```
增量收入 = 激活人数 × 该组历史人均消费 × 折扣率

折扣率 = 0.5（假设优惠券贡献了50%的购买动机，其余为自然需求）
```

### 乘数模型的形式说明

**为什么是乘法模型？**

乘法模型（base × R × F × M）假设 R、F、M 三个因子对响应率的影
响相互独立。这一形式在 RFM 营销文献中被广泛采用（Fader & Hardie, 2009），
优点是可解释性强、运营团队可直接计算。

**局限：**
1. **未考虑交互效应**——例如"高 M + 已流失"用户的响应率可能比各自乘积预测的更低（消费力强但已流失 180+ 天，不应简单地乘以 1.3×0.3）
2. **独立性假设未验证**——R、F、M 在真实数据中通常存在相关性
3. **替代方案未评估**——加法模型（base + R_effect + F_effect + M_effect）或 Logistic 回归可能更准确，但需要历史营销响应数据来校准

在积累足够的营销活动数据后，建议替换为 uplift model 或逻辑回归模型。

行业数据来源：DMA National Client Email Report 2025、Campaign Monitor Benchmarks 2025、Klaviyo Ecommerce Benchmarks 2025。

> **注意**：行业报告通常不按 RFM 分组披露 ROI，以上分组对标仅为量级参考。各行业、各客单价区间差异显著。

### 关键假设与局限性

1. **响应率基准**来自 Campaign Monitor / Klaviyo 2025 电商报告，实际需通过 AB 测试校准
2. **乘数模型**的乘数为定性估计（见上文"乘数模型的形式说明"），来自行业文献而非本项目数据的统计回归。本项目使用模拟数据，不含历史营销响应记录，无法进行数据驱动的乘数校准
3. **增量收入**未考虑用户生命周期价值（LTV）——一个被召回的流失高潜用户可能带来持续复购
4. **优惠券核销率**为行业均值，实际受面额、门槛、品类影响
5. **未计入固定成本**（CRM工具费、设计费）——仅计可变触达成本；已在预算对照表中单独列出
6. **数据为模拟数据**——实际效果可能与预估有显著差异；各组样本量（特别是核心高价值 16 人）限制统计推断精度
7. **各组 ROI 未逐个独立验证**——响应率乘数模型对各组给出的是方向性估计，单组 ROI 点估计的精度因样本量不同而异
